\hypertarget{md__r_e_a_d_m_e_autotoc_md14}{}\doxysection{\texorpdfstring{IoT System for Air Quality Monitoring}{IoT System for Air Quality Monitoring}}\label{md__r_e_a_d_m_e_autotoc_md14}
This project is part of the {\bfseries{Chas Challenge}} and focuses on the IoT system responsible for collecting air quality data using sensors and sending it to the backend for processing and storage.\hypertarget{md__r_e_a_d_m_e_autotoc_md15}{}\doxysubsection{\texorpdfstring{Overview}{Overview}}\label{md__r_e_a_d_m_e_autotoc_md15}
The IoT system is built around an Arduino-\/based microcontroller and various environmental sensors. It collects real-\/time data on air quality, including metrics such as temperature, humidity, pressure, CO₂ levels, and volatile organic compounds (VOCs). The data is transmitted to the backend for further analysis and integration with the mobile app.\hypertarget{md__r_e_a_d_m_e_autotoc_md16}{}\doxysubsection{\texorpdfstring{Hardware Components}{Hardware Components}}\label{md__r_e_a_d_m_e_autotoc_md16}

\begin{DoxyItemize}
\item {\bfseries{Microcontroller}}\+: Arduino UNO REV 4 Wi\+Fi
\item {\bfseries{Sensors}}\+:
\begin{DoxyItemize}
\item Spark\+Fun ENS160/\+BME280 Environmental Combo Breakout
\item SPS30 Particulate Matter (PM) Sensor
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md17}{}\doxysubsection{\texorpdfstring{Features}{Features}}\label{md__r_e_a_d_m_e_autotoc_md17}

\begin{DoxyItemize}
\item Real-\/time air quality monitoring
\item Data transmission to the backend via Wi\+Fi
\item Compensation for temperature and humidity in sensor readings
\item Integration with backend APIs for data storage and analysis
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md18}{}\doxysubsection{\texorpdfstring{How It Works}{How It Works}}\label{md__r_e_a_d_m_e_autotoc_md18}

\begin{DoxyEnumerate}
\item The Arduino collects data from the connected sensors.
\item The data is processed locally, including compensation for environmental factors.
\item The processed data is sent to the backend using HTTP POST requests.
\item The backend stores the data and makes it available for the mobile app.
